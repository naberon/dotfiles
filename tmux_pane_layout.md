# tmux ペイン分割 — エディタ + lazygit/claude レイアウト

## 前提

prefix キー: `C-b`（tmux デフォルト。以前 `C-j` にカスタマイズしていたが、nvim の `Ctrl-j` ウィンドウ移動と衝突するため `C-b` に戻した）

## キーバインド一覧

| 操作 | キー |
|---|---|
| 縦分割（左右に並ぶ） | `C-b` → `\|` |
| 横分割（上下に並ぶ） | `C-b` → `-` |
| ペイン移動: 左 | `C-b` → `h` |
| ペイン移動: 下 | `C-b` → `j` |
| ペイン移動: 上 | `C-b` → `k` |
| ペイン移動: 右 | `C-b` → `l` |
| ペインリサイズ | `Alt` + `←↑↓→`（prefix不要、3セルずつ） |
| ウィンドウ作成 | `C-b` → `c` |
| ウィンドウ切替 | `C-b` → `n` / `p` |
| ペインズーム（現ペインだけ全画面化、再度で解除） | `C-b` → `z` |
| ペインを閉じる | `C-b` → `x`（確認プロンプトで `y`） |

出典: tmux/tmux.conf 64-82行目

## やりたいレイアウトの作り方

左: エディタ（nvim）、右: lazygit / claude / その他 を上下に並べる場合:

1. `C-b` → `|` … 縦分割で右側にペインを作る（今いたペインが左に残る、新ペインは右）
2. 右ペインにカーソルがある状態で `C-b` → `-` … 右ペインを上下に分割
3. 必要なら `C-b` → `-` をもう一度、右ペインをさらに分割（lazygit / claude / その他の3段にする場合）

結果イメージ:

```
+------------------+------------------+
|                  |     lazygit      |
|                  +------------------+
|      nvim        |      claude      |
|                  +------------------+
|                  |     その他       |
+------------------+------------------+
```

ペイン間移動は `C-b` → `h/j/k/l`。

## Tips

- 分割直後のペインは分割元と同じカレントディレクトリで開く
- レイアウトを毎回作り直すのが面倒なら、tmux セッション自体をスクリプト化（`new-session`, `split-window`, `send-keys` をシェルスクリプトにまとめる）するか、tmuxinator / tmuxp のようなセッションマネージャの導入を検討する余地あり（未導入）

## nvim 側の操作

leader キー: `Space`（nvim/lua/keymap.lua:4）

### ウィンドウ分割

カスタムキーマップなし。vim デフォルトのコマンドを使う。

| 操作 | コマンド / キー |
|---|---|
| 横分割（上下に並ぶ） | `:split` または `Ctrl-w` → `s` |
| 縦分割（左右に並ぶ） | `:vsplit` または `Ctrl-w` → `v` |
| 分割を閉じる | `:q` または `Ctrl-w` → `c` |
| 他のウィンドウ以外を閉じる | `Ctrl-w` → `o` |

### ウィンドウ移動

| 操作 | キー |
|---|---|
| 分割ウィンドウ移動: 左 | `Ctrl-h` |
| 分割ウィンドウ移動: 右 | `Ctrl-l` |
| 分割ウィンドウ移動: 下 | `Ctrl-j` |
| 分割ウィンドウ移動: 上 | `Ctrl-k` |

出典: nvim/lua/keymap.lua 35-38行目（`<C-w><C-h>` 等のラップ）

prefix を `C-b` に戻したことで `Ctrl-j` の衝突は解消済み。nvim 内で分割している場合は `Ctrl-h/j/k/l` すべて問題なく使える。

### ウィンドウリサイズ

カスタムキーマップなし。vim デフォルトを使う。

| 操作 | キー |
|---|---|
| 高さ拡大/縮小 | `Ctrl-w` → `+` / `-` |
| 幅拡大/縮小 | `Ctrl-w` → `>` / `<` |
| 全ウィンドウ均等化 | `Ctrl-w` → `=` |

### 現ペインだけ一時表示したい場合

nvim にズームトグル用のプラグイン/キーマップは未導入。2択:

- **tmux ズーム（推奨・可逆）**: `C-b` → `z` で現ペインを全画面化、もう一度で元のレイアウトに復帰。nvim の分割構造はそのまま保持される。ステータスバーに `Z` 表示。
- **`Ctrl-w` → `o`（不可逆）**: 他 window を全部閉じる。元レイアウトに戻すには分割し直しが必要。一時表示用途には不向き。

### 内蔵ターミナル（toggleterm.nvim）

tmux でペインを物理的に分けず、nvim 内でターミナルを開きたい場合の代替手段。

| 操作 | キー |
|---|---|
| ターミナル トグル（フロート） | `Ctrl-\` |
| フロートターミナル | `Space` → `tt` |
| 水平分割ターミナル | `Space` → `th` |
| 垂直分割ターミナル | `Space` → `tv`（幅80） |

出典: nvim/lua/plugins/toggleterm.lua 14-18行目

lazygit をこの中で起動する運用も可能（`:ToggleTerm` → `lazygit`）。ただし現状 lazygit は別コマンド/別ペインとして起動している想定なら、tmux 側のペイン分割で足りる。
